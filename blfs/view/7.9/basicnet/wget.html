Wget-1.17.1

Introduction to Wget

The Wget package contains a utility useful for non-interactive downloading of files from the Web.

This package is known to build and work properly using an LFS-7.9 platform.

Package Information

  • Download (HTTP): http://ftp.gnu.org/gnu/wget/wget-1.17.1.tar.xz

  • Download (FTP): ftp://ftp.gnu.org/gnu/wget/wget-1.17.1.tar.xz

  • Download MD5 sum: b0d58ef4963690e71effba24c105ed52

  • Download size: 1.8 MB

  • Estimated disk space required: 28 MB (with tests)

  • Estimated build time: 0.5 SBU (with tests)

Wget Dependencies

Recommended

GnuTLS-3.4.9

Optional

libidn-1.32, OpenSSL-1.0.2g, PCRE-8.38, Valgrind-3.11.0 (optional for the test suite), IO::Socket::SSL, HTTP::Daemon (both required for the test suite), and libpsl

User Notes: http://wiki.linuxfromscratch.org/blfs/wiki/wget

Installation of Wget

Install Wget by running the following commands: 

./configure --prefix=/usr      \
            --sysconfdir=/etc  &&
make

To test the results, issue: make check. The HTTPS tests fail if openssl is used and valgrind is enabled.

Now, as the root user: 

make install

Command Explanations

--sysconfdir=/etc: This relocates the configuration file from /usr/etc to /etc.

--with-ssl=openssl: This allows the program to use OpenSSL-1.0.2g instead of GnuTLS-3.4.9.

--enable-valgrind-tests: This allows the tests to be run under valgrind.

Configuring Wget

Config Files

/etc/wgetrc and ~/.wgetrc

If you have installed the Certificate Authority Certificates and you want Wget to use them, as the root user: 

echo ca-directory=/etc/ssl/certs >> /etc/wgetrc

Contents

Installed Program: wget
Installed Libraries: None
Installed Directories: None

Short Descriptions

wget

retrieves files from the Web using the HTTP, HTTPS and FTP protocols. It is designed to be non-interactive, for background or unattended operations.
